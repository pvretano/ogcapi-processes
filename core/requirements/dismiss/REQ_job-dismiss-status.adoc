[[req_dismiss_job-dismiss-status]]
[requirement]
====
[%metadata]
identifier:: /req/dismiss/job-dismiss-status

[.component,class=part]
--
If the job is currently in the `accepted` or `running` state, then the server SHALL set the status of the job to `dismissed`.
--

[.component,class=part]
--
If the job is currently in the `successful`, `failed` or `dismissed` state, then the server SHALL remove the job and it SHALL no longer be accessible via the Processes API.
--

====

NOTE: This standard makes no statements regarding the disposition of artifacts created by a job once a job has been removed and is no longer accessible via the Processes API.
